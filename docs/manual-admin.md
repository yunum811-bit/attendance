## 1. ภาพรวมสิทธิ์ตาม Role

| Role | สิทธิ์ |
|---|---|
| **employee** (พนักงาน) | Check In/Out, ขอลา, ดูรายงานตัวเอง |
| **manager** (หัวหน้าแผนก) | + อนุมัติลาพนักงานในแผนก, ดูรายงานแผนก |
| **md** (กรรมการผู้จัดการ) | + อนุมัติลาหัวหน้าแผนก, ดูรายงานทั้งหมด |
| **admin** (ผู้ดูแลระบบ) | + จัดการพนักงาน, ตั้งค่าระบบ, ทุกอย่าง |
| **manager_admin** (หัวหน้า+Admin) | สิทธิ์ Manager + Admin รวมกัน |

## 2. จัดการพนักงาน

เมนู: **"👥 พนักงาน"**

### เพิ่มพนักงานใหม่:
1. กด **"+ เพิ่มพนักงาน"**
2. กรอก: รหัสพนักงาน, ชื่อ, นามสกุล, อีเมล, แผนก, บทบาท, รหัสผ่าน
3. กด **"บันทึก"**

> รหัสพนักงานต้องไม่ซ้ำ และแก้ไขไม่ได้ภายหลัง

### แก้ไขข้อมูลพนักงาน:
1. กดปุ่ม **"✏️ แก้ไข"** ที่แถวพนักงาน
2. แก้ไข: ชื่อ, นามสกุล, อีเมล, แผนก, บทบาท
3. กด **"💾 บันทึก"**

> รหัสพนักงาน (Employee Code) แก้ไขไม่ได้

### จัดการรหัสผ่าน:
- **"🔑 รหัสผ่าน"** — ตั้งรหัสผ่านใหม่ให้พนักงาน
- **"🔄 Reset"** — รีเซ็ตเป็น "1234"

## 3. อนุมัติการลา

เมนู: **"✅ อนุมัติ"**

### ช่อง "คำขอลาจากพนักงาน":
- แสดงคำขอจากพนักงานทุกแผนก (สำหรับ Admin)
- รวมพนักงานที่ไม่มีหัวหน้าแผนก
- กด **"✅ อนุมัติ"** หรือ **"❌ ปฏิเสธ"** (ต้องระบุเหตุผล)

### ช่อง "คำขอลาจากหัวหน้าแผนก" (MD/Admin Only):
- แสดงคำขอจากหัวหน้าแผนกทุกแผนก
- เฉพาะ MD หรือ Admin อนุมัติได้

### ลำดับการอนุมัติ:
```
พนักงาน → หัวหน้าแผนก อนุมัติ
พนักงาน (ไม่มีหัวหน้า) → Admin/MD อนุมัติ
หัวหน้าแผนก → MD/Admin อนุมัติ
```

## 4. จัดการประเภทการลา

เมนู: **"📝 ประเภทการลา"**

### เพิ่มประเภทใหม่:
1. กด **"+ เพิ่มประเภทการลา"**
2. กรอก: ชื่อ, จำนวนวันสูงสุด/ปี, คำอธิบาย
3. กด **"💾 เพิ่มประเภทการลา"**

### แก้ไข/ลบ:
- **"✏️ แก้ไข"** — เปลี่ยนชื่อ, จำนวนวัน
- **"🗑️ ลบ"** — ลบได้เฉพาะประเภทที่ไม่มีคนใช้

### ประเภทเริ่มต้น:
| ประเภท | วัน/ปี |
|---|---|
| ลาป่วย | 30 |
| ลากิจ | 5 |
| ลาพักร้อน | 10 |
| ลาคลอด | 98 |

## 5. กำหนดวันลารายบุคคล

เมนู: **"📅 กำหนดวันลา"**

กำหนดวันลาเฉพาะบุคคล (ไม่เท่ากัน):

1. เลือกปี
2. กดปุ่ม **"✏️ แก้ไข"** ที่พนักงานที่ต้องการ
3. ใส่จำนวนวันลาแต่ละประเภท
4. ถ้าเว้นว่าง = ใช้ค่าเริ่มต้นของระบบ
5. กด **"💾 บันทึก"**

> ตัวอย่าง: พนักงานใหม่อาจได้ลาพักร้อน 5 วัน แทน 10 วัน

## 6. สร้างประกาศ

เมนู: **"📢 ประกาศ"**

### สร้างประกาศใหม่:
1. กด **"+ สร้างประกาศ"**
2. กรอก: หัวข้อ, เนื้อหา
3. แนบไฟล์ (ถ้ามี) — รองรับ PDF, Word, Excel, รูปภาพ (สูงสุด 5MB)
4. กด **"📢 เผยแพร่ประกาศ"**

### ลบประกาศ:
- กดไอคอน **"🗑️"** ที่มุมขวาของประกาศ

> ประกาศจะแสดงให้พนักงานทุกคนเห็น

## 7. ดูรายงาน + Export

เมนู: **"📊 รายงาน"**

### ประเภทรายงาน:
1. **รายงานการเข้างาน** — วันที่, เวลาเข้า/ออก, ตำแหน่ง GPS
2. **รายงานการลา** — ประเภท, วันที่, สถานะ, ผู้อนุมัติ
3. **สรุปรายเดือน** — จำนวนวันมาทำงาน, มาสาย, ลา

### กรองข้อมูล:
- Admin เห็นทุกแผนก + เลือกกรองตามแผนกได้
- เลือกช่วงวันที่ / เดือน

### Export:
- กด **"📄 Export CSV"** — เปิดใน Excel ได้
- กด **"📊 Export XLSX"** — ไฟล์ Excel โดยตรง

## 8. ตั้งค่าบริษัท

เมนู: **"⚙️ ตั้งค่า"**

### ชื่อบริษัท:
1. พิมพ์ชื่อบริษัท
2. กด **"💾 บันทึกชื่อ"**
3. จะแสดงที่ Header + หน้า Login

### โลโก้บริษัท:
1. กด **"เลือกไฟล์โลโก้"**
2. เลือกรูป (PNG, JPG, GIF, SVG สูงสุด 2MB)
3. กด **"📤 อัปโหลดโลโก้"**
4. จะแสดงที่ Header + หน้า Login

---

## 9. จัดการรหัสผ่าน

### ตั้งรหัสผ่านให้พนักงาน:
1. ไปที่ "👥 พนักงาน"
2. กด **"🔑 รหัสผ่าน"** ที่แถวพนักงาน
3. ใส่รหัสผ่านใหม่ (อย่างน้อย 4 ตัว)
4. กด **"💾 บันทึกรหัสผ่าน"**

### Reset เป็นค่าเริ่มต้น:
- กด **"🔄 Reset"** → รหัสผ่านจะเป็น "1234"

---

## 10. ระบบแจ้งเตือน

Admin จะได้รับแจ้งเตือนเมื่อ:
- 📋 พนักงานขอลา (ทุกแผนก)
- 📋 หัวหน้าแผนกขอลา
- 📋 พนักงานที่ไม่มีหัวหน้าแผนกขอลา

กดไอคอน **🔔** ที่ Header เพื่อดูแจ้งเตือน

## 12. การ Deploy & Maintenance

### อัปเดตระบบ:
```bash
# บนเครื่องตัวเอง
git add . && git commit -m "update" && git push

# Railway จะ deploy อัตโนมัติ
```

### ข้อมูลสำคัญที่ต้องไม่หาย:
- Database: อยู่ใน Volume `/data/attendance.db`
- รูปถ่าย: อยู่ใน Volume `/data/uploads/`
- โลโก้: อยู่ใน Volume `/data/uploads/`

### Environment Variables (Railway):
| Key | Value |
|---|---|
| DATA_DIR | /data |
| UPLOADS_DIR | /data/uploads |

### Backup:
```bash
# SSH เข้า server (ถ้าใช้ VPS)
bash backup.sh
```

### ดู Logs:
- Railway: แท็บ Deployments → View Logs
- VPS: `pm2 logs attendance`
